# 申论讲义一站式处理（拆分 + 标注 + PDF）

这个 skill 把一份含多讲多题的申论讲义 MD，**两阶段**处理成可直接学习的材料：

1. **阶段一 · 拆分**：大 MD → 每题一个独立 MD
2. **阶段二 · 标注**：读阶段一产物 → 给每个 MD 加踩点标注
3. **阶段三 · 渲染**：两种 MD 都转成 PDF

**关键顺序**：**必须先拆完、让用户确认文件清单无误，再进入标注阶段**。标注是对已拆出来的独立 MD 做的，不是对大讲义做的。

## 输出目录结构

```
输出根目录/
├── 讲义_01_xxx.md              ← 阶段一产物（基础版）
├── 讲义_02_xxx.md
├── ……
├── PDF/                         ← 阶段三：基础版 PDF
│   ├── 讲义_01_xxx.pdf
│   └── ……
├── 标注md/                      ← 阶段二产物（标注版）
│   ├── 标注版_讲义_01_xxx.md
│   └── ……
└── 标注md/PDF/                  ← 阶段三：标注版 PDF
    ├── 标注版_讲义_01_xxx.pdf
    └── ……
```

# 阶段一 · 拆分讲义

## 核心原则（踩过的坑）

1. **题目识别不能只看「真题/试题」**。申论讲义里题目块的三级标题至少有：`### 题目`、`### 真题 1/2`、`### 试题 1/2/3`，**都要覆盖**。否则「一讲一题」的讲会被整讲漏掉。
2. **共享材料要合并**，不能把共享材料的题目各拆一份（否则同段材料重复抽出，最后一题只剩题干）。
3. **文章写作（大作文）讲结构特殊**：用 `#### 给定资料X` 标记多段材料，可能有 `同第X讲题目材料` 这种跨讲引用，题干在最后。必须把所有给定资料段**按顺序拼接**作为材料，跨讲引用去源讲抓回来。
4. **不要漏讲**。先打印所有「讲」和「题目候选行」给用户对齐，确认每讲应输出几个文件，再动手。

## 第一步：读源文件

```python
import sys
sys.stdout.reconfigure(encoding='utf-8')

with open(source_path, 'r', encoding='utf-8') as f:
    lines = f.read().split('\n')
```

必须 `encoding='utf-8'`，Windows 下不指定会乱码。

## 第二步：扫描结构（全量、分类）

```python
import re
lectures, subheaders, questions, share_markers, data_headers = [], [], [], [], []
for i, line in enumerate(lines, start=1):
    s = line.strip()
    if re.match(r'^##\s*第\s*[\d\-\\]+\s*讲', s):
        lectures.append((i, s))
    if re.match(r'^###\s*(真题|试题|题目)', s):
        subheaders.append((i, s))
    if s.startswith('**') and ('根据' in s or '结合' in s):
        questions.append((i, s))
    if re.search(r'同(试题|上|第)[^，。\s]*材料', s):
        share_markers.append((i, s))
    if re.match(r'^####\s*给定(资料|材料)', s):
        data_headers.append((i, s))
```

**先把 5 类标记打印给用户**，对齐「这份讲义应拆成 N 个文件」。

## 第三步：判断文件数（4 种模式）

对每个「讲」：

### 模式 A — 一讲一题
只有 1 个 `### 题目` / 真题 / 试题。→ **1 个文件**

### 模式 B — 一讲多题，材料独立
多个子题且每个子题下都有自己独立的材料段。→ **每个子题 1 个文件**

### 模式 C — 一讲多题，材料共享
出现「同试题1材料」「同上材料」标记。→ **整讲合并 1 个文件**。题型名按**第一道题**的题型写（即使后面子题题型不同也不拼接）。

### 模式 D — 大作文（多段 `#### 给定资料X`）
→ **1 个文件**（议论文），材料 = 所有给定资料段按原顺序拼接；遇到 `同第X讲题目材料` 默认自动回溯到第 X 讲抓材料原文塞入，**无需逐次确认**。

### 判断逻辑

```
data = 该讲内 #### 给定资料X 段数
subs = 该讲内 ### 真题/试题/题目 数
if data >= 2: → 模式 D
elif subs <= 1: → 模式 A
elif 出现 share_markers: → 模式 C
else: → 模式 B（拆 subs 份）
```

**动手前必须列出 N 个文件清单（序号/题型/主题）给用户确认。**

## 第四步：提取内容

- **材料起点**：题目块小节标题的下一行起，跳过空行和 `#` 开头，取第一个实际内容行
- **材料终点**：题干行前一行
- **题干**：从 `**根据…**` / `**请…**` 起，收到 `---` 或下一个 `#` 或空白分隔符停止
- **小节下无独立新材料时**（如第 6 讲·真题2 这种）：材料 = 前一题题干结束到本题题干之间的所有正文，"与前一题的间隔整段"算本题材料，不要越界到前一题材料区

## 第五步：命名

`{原文档大标题}_{两位数序号}_{题型}_{主题}.md`

- 原文档大标题：源文件首个 `# 标题`
- 序号：**跨讲连续编号**，两位数
- 题型：归纳概括/提出对策/综合分析/应用文/议论文（模式 D 一律"议论文"）
- 主题：5-10 字概括

# 阶段二 · 标注

**前提**：阶段一已完成，`输出根目录/` 下有 N 个独立题目 MD。标注是对拆出来的独立 MD 做的。

## 一条核心原则

**注解不是给原文贴"语法标签"，而是在告诉答题者"这句话该怎么用"。**

反面教材（禁用）：`【时间数据】`、`【年份数据】`、`【人名】`、`【数字】` 这种纯语法标签没有任何答题价值。
正面写法：`【立法周期数据，体现立法严谨性】`、`【群众感受原文，关键词：距离拉近】`——每条都指向答题用途。

## 标注三要素（每条 `【】` 必须回答至少一个）

1. **它是什么**（问题/对策/效果/背景/本质/案例…）
2. **答题怎么用**（核心得分点/需巩固/强化方向/非得分点/开头背景）
3. **踩点提示**（关键词原词/不可替换/编号对应）

对比示例：

| 坏 | 好 |
|---|---|
| `【时间】` | `【时间节点+政策变化背景】` |
| `【数据】` | `【立法周期数据，体现立法严谨性】` |
| `【案例】` | `【案例一：小祝，论证平台成效】` |
| `【措施】` | `【⭐核心经验2：迅速反应+组织专家解读+"一对一"指导】` |

## 八大标注类型

### 1. 核心论点类（⭐ 开头，必须编号）
- `【⭐核心经验1：xxx】` / `【⭐核心对策1：xxx】` / `【⭐核心问题1：xxx】`
- `【核心观点句：xxx，评分硬通货】` / `【本质揭示：xxx，核心得分点】`
- 全文**至少 1 个 ⭐**，同类**连续编号**（经验1→2→3）

### 2. 问题 / 表现类（分层标）
- `【核心问题一：责任主体不明】`
- `【问题一表现①：政府层面——通车后不养不管】`
- `【问题一表现②：村级层面——制度缺失】`
- 有核心问题就拆 ①②③ 表现

### 3. 对策 / 措施类（"措施 + 结果"双轨成对）
- `【措施-干部转变：学电脑、学讲话、交流心得、实战锻炼】`
- `【结果-干部转变：从怕事躲避到主动担当】`
- 成对出现，体现"措施→结果"的得分结构

### 4. 现有举措编号类（答"要巩固什么"题专用）
- `【现有举措1：平台名称，需巩固，关键词"集成平台"】`
- `【现有举措2：制度创新，需巩固】`
- 连续编号 + **必带"需巩固"** 标记

### 5. 强化方向类（答"下一步怎么办"题专用）
- `【强化方向：简化网上操作流程（源于小李"操作麻烦点"的反馈）】`
- **必须指出依据来源**（哪句话推出的），不凭空下结论

### 6. 背景 / 引入类（非得分点警示）
- `【场景引入，非直接得分点】`
- `【背景：xxx，非直接得分点】`
- `【背景关键词：必须写入开头段】`
- 必须明确"是否直接得分"，防止误写入答案主体

### 7. 效果 / 成效 / 验证类
- `【效果：不出村学戏】`（抽象效果）
- `【效果验证：3人→30人，证明留得住】`（具体数据验证）
- 抽象效果 + 具体数据**两者都要标**

### 8. 评分 / 易错 / 踩点提示类
- `【必须使用原词，不可替换为"权责不清"或"管理缺位"】`
- `【将"两尺"转化为"窄"，保留核心语义即可得分】`
- `【2分】` / `【1分】`（默认不加，用户要求才加）
- `【易错警示：这里的"气候"指商业氛围……切勿归为"有形"硬件】`

## 进阶技巧

### 技巧 A：群众反馈双向拆解
群众/用户原话引述时，**前后两半各标一次**：
> "虽然在网上操作麻烦点，但与专程回老家相比，还是省事多了。"【用户反馈：小李的原话，前半句"操作麻烦"直接指向强化方向，后半句肯定成效】

### 技巧 B：段落末尾"核心信息提取"块（长段落可选）
长且信息密集的段落末尾追加速查卡：
```
【段落核心信息提取】
• 服务对象：普通群众【分类一】
• 巩固成果：集成办理平台、"以数治税"、"云税驿站"、"税管家"、多部门联办
• 强化方向：简化网上操作流程
```
不是每段都要，只在信息密度高时用。

### 技巧 C：表层 vs 核心对比
- `【表层现象：从观众到爱好者】`（叙述铺陈）
- `【要点1核心对策：结对子、育骨干】`（答案关键）
明确区分"铺陈"和"得分点"，避免把表层当答案。

### 技巧 D：原词必须保留的提示
遇材料金句时，标 `【关键词需原词呈现："xxx"】`。

## 加粗规则（严格）

**只给这 5 类词加粗**：
1. 核心动词（"关停"、"打通"、"构建"）
2. 核心名词短语（"集成办理平台"、"云税驿站"、"结对子、育骨干"）
3. 具体数据（"50余家"、"3人到30人"、"100多场"）
4. 专有名词 / 政策名词（"《电子商务法》"、"以数治税"、"一枚印章管审批"）
5. 原词金句（"农民演、农民看"、"没有落后的行业"）

**不加粗**：时间、过渡词、连词、形容词堆砌、情感描写。

## 质量底线（违反视为未完成，标完自检）

每份标注必须同时满足：
- [ ] **⭐ 核心标记 ≥ 1 个**（核心经验/对策/问题/观点）
- [ ] **"非直接得分点" 警示 ≥ 1 处**（必有背景段）
- [ ] **8 类注解覆盖 ≥ 5 类**
- [ ] **加粗 ≥ 每段 1 处**（纯背景段可例外）
- [ ] **编号类一致**（经验1/2/3、问题一/二/三、举措1/2/3 成体系）
- [ ] **禁用纯语法标签**（`【时间】`、`【数据】`、`【人名】` 单独出现）
- [ ] **自检题**：不看原文只看标注，能不能列出答案要点？不能就重标。

## 开工顺序（硬性）

1. 读源 MD（`encoding='utf-8'`）
2. **先看题干**——题问什么决定标什么：
   - 问"经验"→ 重点标 ⭐核心经验
   - 问"问题"→ 重点标核心问题 + 表现①②③
   - 问"对策/巩固"→ 重点标现有举措N + 强化方向
   - 问"认识/启示"→ 重点标核心观点句 + 本质揭示
3. **按段标，不跨段**
4. 长段落加 `【段落核心信息提取】` 块
5. **跑质量底线 7 项自检**
6. 输出到 `输出根目录/标注md/标注版_{原名}.md`

## 核心动作提醒

**只做增量标注，不改原文**：只能加粗和追加 `【…】`，不要删字、改字、调整语序。

## ⚠️ 必须使用「手动精标」，禁用「自动批量标注」

**实测对比**：手动精标 01-05 质量明显高于一次性自动批标。原因：
- 自动批标倾向于重复同类标签、偷懒堆砌、漏掉题干导向的重点
- 手动精标能按题干类型针对性抓点、控制密度、保持编号一致性

**强制工作方式**：

1. **一次只处理一个 MD 文件**，读完原文、想透题干要什么要点，再下笔标注
2. **不要写一个"批量标注脚本"**——机械正则只能替换，无法判断"这句是得分点还是背景"
3. **不要并行生成多份标注**，一份一份来
4. **每标完一份都跑一遍"质量底线 7 项自检"**，过不了就当场重标，不要拖到最后
5. 遇到拿不准的句子（这算得分点还是背景？）**停下来问用户**，不要硬标

违反这条规则的最典型症状：出现大量只有 `【时间数据】`、`【数据】`、`【人名】` 这种纯语法标签 → 说明在机械批标，立即停下重新手动精标。

## 标注的正确打开方式

1. **先拿第一个文件手动精标完给用户看**，确认风格 OK 再逐个处理其余的（仍然是手动精标，不是批处理）
2. 用户若给了参考样例，先读 1-2 个对齐风格再开工
3. 分值提示（`【2分】`）默认不加

# 阶段三 · 渲染 PDF

用 Playwright（**不要用 pandoc**，中文支持差）：

```bash
pip install playwright markdown
playwright install chromium
```

## 基础版 PDF

```bash
python scripts/md_to_pdf.py <输出根目录> <输出根目录>/PDF
```

## 标注版 PDF

```bash
python scripts/annotate_to_pdf.py <输出根目录>/标注md <输出根目录>/标注md/PDF
```

标注版脚本会自动把文中 `【…】` 替换成 `<span class="annot">` 灰底楷体呈现，正文仍是雅黑，加粗红色。

## 版式规格（已调小到标准教材级）

- 正文 10.5pt（五号字），行距 1.5
- H1 15pt / H2 12.5pt / H3 11pt
- 注解 9pt 灰底楷体
- 页边距 18mm / 16mm

# 全流程检查清单

开工前与用户对齐：
1. 源文件路径、输出根目录
2. 打印扫描结果（5 类标记），对齐讲义结构
3. 列出拆分后的 N 个文件清单（序号/题型/主题），等用户 ✅
4. 是否需要标注阶段（默认做，用户可跳过）
5. 是否需要分值提示 `【2分】`（默认不加）

流程：
1. 拆出 N 个基础 MD → 确认文件清单
2. **拿第一个 MD 做标注样本** → 让用户确认风格
3. 批量标注其余 MD → 输出到 `标注md/`
4. 渲染基础版 PDF → `PDF/`
5. 渲染标注版 PDF → `标注md/PDF/`

# 常见坑汇总

**拆分阶段：**
- 编码：Python 读写都要 `encoding='utf-8'`；Windows 终端加 `sys.stdout.reconfigure(encoding='utf-8')`
- 路径：Windows 用原始字符串 `r'C:\...'` 或正斜杠
- `### 题目` 不要漏（和真题/试题一样是题目块起点）
- 共享材料题不重复拆（模式 C 整讲合并）
- 大作文 `同第X讲题目材料` 要回溯原文
- 题目序号跨讲连续

**标注阶段：**
- 不要改原文，只加粗 + 追加 `【…】`
- 非得分背景必须标，避免学习者误写入答案
- 7 类注解要覆盖，不要只堆一类
- 先做第一个给用户确认风格再批量

**PDF 阶段：**
- 不用 pandoc，用 Playwright
- `【】` 样式走 `.annot` class 渲染，灰底楷体
- 基础版 PDF → `PDF/`，标注版 PDF → `标注md/PDF/`
